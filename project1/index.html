CSS grid & layout

In this page, CSS Flexbox was chosen as an appropriate solution for the images. The same results could have been implemented with grid as well, but due to the small number of elements in the container, Flexbox was more appropriate. With more items, using grid to further customize placement of each elements might give a cleaner and more refined look.

On the other hand, grid was used to position elements in the document section. Experience with both layout types will be useful for further development.

To ensure that the content is readable on smaller computer screens, a media query is used that changes the layout from grid to block and flex-row to flex-column.

jQuery

jQuery is a Javascript library that makes DOM manipulation, animation and event handling easier to write in vanilla JS. In this web page, jQuery has been used in the files canvas.js and main.js. These files have been imported at top of the HTML document, as the script will not run before the document is ready (due to jQuery $(document).ready(function () {...} which encapsulates the code.) In jQuery, you can access each element in the HTML document by accessing their selector. You access the selector by writing $("selector"). E.g., $(".wrapper").click(function () {...}, which adds an event listener to html elements with the class "wrapper", that listens for the "click" event.

In this page, jQuery is imported using the online version. It is therefore required to be connected to the Internet in order to view this web page.

HTML Canvas

HTML Canvas is an element which can be accessed with Javascript to draw figures. Most of the animation and scripting is done in canvas.js. The figure is drawn in the drawCanvas() function, while the animation is executed in the animate() function.

SVG & CSS animation

The SVG is drawn in the HTML body. The animation is done via css animation. Due to time restriction, having the circle bouncing around randomly was not achievable.

Testing

To ensure cross-browser compatability, the website has been tested in Safari, Google Chrome and Mozilla Firefox. All browsers displayed similar behaviour.
